
# Руководство по E2E тестированию в SpriteForge

E2E тесты в нашем приложении — это автоматизированные сценарии, которые имитируют действия пользователя (движения мыши, клики, драг-энд-дроп). Они запускаются через меню **System Diagnostics (иконка жука)**.

## Правило №1: Один файл — один сценарий

Чтобы тесты были надежными и легко читаемыми, мы используем модульную структуру:
- **`tests/simulator.ts`**: Ядро симулятора. Общие методы для кликов, перетаскивания и ожидания элементов.
- **`tests/scenarios/*.ts`**: Сами тесты. Каждый файл содержит ровно один сценарий.
- **`tests/e2e.spec.ts`**: Агрегатор, где все тесты собираются в один список.

**Если вы добавляете новый тест, создайте новый файл в `tests/scenarios/` и импортируйте его в `tests/e2e.spec.ts`.**

---

## Правило №2: Используйте `data-testid`

Никогда не привязывайте тесты к CSS-классам или текстовому содержимому. 
**Всегда добавляйте `data-testid` к элементам, с которыми должен взаимодействовать тест.**

---

## Анатомия сценария (`tests/scenarios/...`)

Каждый сценарий состоит из двух фаз:
1. **Setup**: Подготовка (сброс проекта, загрузка тестовых картинок). Если setup падает, тест останавливается.
2. **Steps**: Последовательность действий для проверки фичи.

```typescript
import { Simulator, E2EScenario } from '../simulator';

export const my_test: E2EScenario = {
    id: 'unique_id',
    name: 'Название теста',
    description: 'Что мы проверяем',
    setup: [ ... ],
    steps: [ ... ]
}
```

---

## Возможности Симулятора (`Simulator`)

| Метод | Описание |
|-------|----------|
| `waitFor(selector)` | Ждет появления **видимого** элемента (лучше всего `[data-testid="..."]`). |
| `click(selector)` | Перемещает курсор в центр и имитирует клик. |
| `drag(selector, dx, dy)` | Перемещает элемент на указанное количество пикселей. |
| `type(selector, text)` | Очищает поле и печатает текст. |
| `injectTestImage(name, bg, text, w, h)` | Генерирует картинку с цифрами 1-4 в квадрантах. |
